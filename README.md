# Azyet Fine Cuts 

A professional web presence for **Azyet Fine Cuts**, a premium barber shop specializing in classic grooming and modern styling.

## Project Overview

This project is a multi-page static website designed to showcase shop services, share grooming expertise via a blog, and facilitate customer bookings.

## 🏗️ Project Structure
- `index.html`: The landing page featuring a hero section and service summary.
- `about.html`: Detailed information about the shop's history and team expertise.
- `blog.html`: A content hub for grooming tips and lifestyle articles.
- `contact.html`: Business hours, location info, and a booking inquiry form.

## Getting started
1. Clone this repository on your local machine.

```bash
git clone https://github.com/Kwanusu/azyet_fine_cuts.git

cd azyet_fine_cuts
```
2. Open `index.html` in any modern browser to view the site.
3. No build tools or dependencies required for this static version.

## Tech Stack
- **HTML5**: Semantic structure for accessibility and SEO.
- **Markdown**: For project documentation.

## How to collaborate 
We welcome contributions! To maintain code quality and project integrity, please follow these steps

### The workflow
- **Fork the Repository:** Create your own copy  of the project to work on.
- **Create a Feature Branch:** 
```bash
git checkout -b feature/YourFeatureName
```
- **Commit Your Changes:** Use descriptive commit messages (e.g., feat: add social media icons to footer).
- **Push to branch:** 
```bash 
git push origin feature/YourFeatureName 
```
- **Open a Pull Request(PR):** Describe your changesclearly and link the related issues.

2. **Contribution Guidelines**
Maintain Semantics: Keep the HTML structure semantic. Avoid using `<div>` where `<section>`, `<article>`, or `<nav>` is more appropriate.

**Code Style**: Ensure all tags are properly closed and indented using 2 or 4 spaces.

**Documentation**: If you add a new page or a significant feature, update the "Project Structure" section of this README.

3. **Reporting Issues**
If you find a bug or have a suggestion:

Check the Issues tab to see if it has already been reported.

If not, open a new issue with a clear title and a detailed description of the problem or feature request.
